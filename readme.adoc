= asciidoc

Grammar and typed AST parser for https://docs.asciidoctor.org/[AsciiDoc].

== Overview

* An AsciiDoc document is a collection of top-level https://docs.asciidoctor.org[attributes] and blocks.
* The https://docs.asciidoctor.org/asciidoc/latest/blocks/#content-model[content model] is generally respected and modeled in the AST

== Usage

Parse a complete document with `parse`. The parser returns a discriminated result instead of throwing for invalid input.

[source,ts]
----
import { parse } from "asciidoc";

const result = parse(`= Document title

An opening paragraph with *bold text*.

* First item
* Second item`);

if (!result.ok) {
  throw new Error(result.value.message);
}

for (const block of result.value.blocks) {
  console.log(block.type, block.context);
}
----

On success, `value` is a typed `Document` whose `blocks` contain hydrated nodes such as `Header`, `BlockParagraph`, `BlockList`, and `BlockSource`. Inline markup is represented by typed nodes inside each block's `content`.

To parse a grammar fragment instead of a complete document, pass an Ohm start rule as the second argument:

[source,ts]
----
const header = parse("== Section title", "Header");
const block = parse("NOTE: Remember the milk.", "Block");
----

Use `createParser` when you need an isolated parser instance or runtime AST guards:

[source,ts]
----
import { createParser } from "asciidoc";

const parser = createParser();
const result = parser.parse("A paragraph.");
----
